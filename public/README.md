# 自定义报警声音配置说明

## 如何使用自定义报警声音

系统默认使用生成的报警声音，但您可以使用自己的声音文件来替代默认声音。

### 步骤：

1. **准备您的声音文件**
   - 支持的格式：MP3, WAV, OGG
   - 建议使用响亮、急促、容易引起注意的声音
   - 建议时长：2-5秒

2. **放置声音文件**
   - 将您的声音文件复制到项目的 `public` 目录
   - 将文件命名为 `alarm.mp3`（或您选择的名称）

3. **（可选）自定义声音文件路径**
   - 如果您想使用不同的文件名，需要修改代码
   - 打开 `src/pages/AlarmManagement.tsx`
   - 找到 `useAlarmSound(hasUnprocessedAlarms)` 这一行
   - 改为 `useAlarmSound(hasUnprocessedAlarms, '/您的文件名.mp3')`

### 声音特性：

- **音量**：系统会将音量设置为80%（0.8），确保声音足够响亮
- **循环播放**：当有未处理的报警时，声音会持续循环播放
- **自动停止**：当所有报警都被处理后，声音会自动停止

### 如果找不到自定义声音文件：

系统会自动回退到生成的报警声音，该声音具有以下特点：
- 频率在600Hz和800Hz之间快速切换
- 每200毫秒切换一次，产生急促感
- 持续2秒后短暂停顿，然后重复

### 示例文件结构：

```
well-liquid-diagnosis/
├── public/
│   ├── alarm.mp3          # 您的自定义报警声音
│   └── README.md
├── src/
│   ├── hooks/
│   │   └── useAlarmSound.ts
│   └── ...
└── ...
```

## 报警触发逻辑

- 报警声音会在有**未处理的预警**时自动播放
- 点击"处理"按钮后，该预警会被标记为已处理
- 当所有预警都被处理后，报警声音会自动停止
