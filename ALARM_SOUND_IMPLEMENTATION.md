# 报警声音功能实现说明

## 功能概述

本次更新实现了以下功能：

1. **自动报警声音播放**：当系统存在未处理的预警时，自动播放报警声音
2. **自定义声音支持**：支持使用自己的声音文件替代默认生成的报警声音
3. **响亮且急促的报警声**：默认生成的报警声具有高音量和急促的特点，容易引起注意
4. **处理预警功能**：点击"处理"按钮后，预警状态更新为"已处理"
5. **自动停止报警**：当所有预警都被处理后，报警声音自动停止

## 实现细节

### 1. 报警声音系统

创建了 `useAlarmSound` 自定义 Hook (`src/hooks/useAlarmSound.ts`)，提供以下特性：

- **双模式支持**：
  - 优先使用自定义音频文件（如果存在）
  - 回退到生成的报警声音（如果找不到自定义文件）

- **生成的报警声音特点**：
  - 频率在 600Hz 和 800Hz 之间快速切换
  - 每 200 毫秒切换一次，产生急促感
  - 音量设置为 70% (0.7)
  - 持续 2 秒后短暂停顿（300ms），然后重复

- **自定义音频特点**：
  - 支持 MP3、WAV、OGG 格式
  - 音量设置为 80% (0.8)，确保声音足够响亮
  - 循环播放

### 2. 处理预警功能

- 添加了 `processAlarm` API 函数 (`src/services/api.ts`)
- 点击"处理"按钮后：
  - 更新预警状态为"已处理"
  - 记录处理时间
  - 刷新预警列表
  - 显示成功消息
  - 移除该预警的"处理"按钮

### 3. 报警触发逻辑

在 `AlarmManagement` 组件中：
- 检查是否存在未处理的预警
- 如果有未处理的预警，触发报警声音
- 当所有预警都被处理后，自动停止报警声音

## 如何使用自定义报警声音

### 步骤 1: 准备声音文件

- 选择一个响亮、急促、容易引起注意的声音
- 支持的格式：MP3、WAV、OGG
- 建议时长：2-5 秒

### 步骤 2: 放置声音文件

将声音文件复制到 `public` 目录，并命名为 `alarm.mp3`：

```
well-liquid-diagnosis/
├── public/
│   ├── alarm.mp3          # 您的自定义报警声音
│   └── README.md
└── ...
```

### 步骤 3: (可选) 自定义文件路径

如果想使用不同的文件名，可以修改 `src/pages/AlarmManagement.tsx`：

```typescript
// 当前代码
useAlarmSound(hasUnprocessedAlarms);

// 修改为
useAlarmSound(hasUnprocessedAlarms, '/您的文件名.mp3');
```

## 注意事项

1. **浏览器自动播放策略**：某些浏览器可能会阻止自动播放音频，首次需要用户交互后才能播放
2. **音频文件格式**：建议使用 MP3 格式以获得最佳兼容性
3. **音量控制**：可以通过修改 `useAlarmSound.ts` 中的 `volume` 和 `gain.value` 参数调整音量
